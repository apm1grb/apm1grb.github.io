GOF Pattern

Template Method

In class libraries with similar sibling classes the Template Method pattern is often seen. It allows to define in the parent class a common overall logic, where the individual steps could be adjusted regarding the needs in the derived classes.

In the sample AddDevice is the Template Method which defines three steps. The middle step AddDeviceCore should allow the derived classes to implement its own logic. So this step can be defined as abstract or virtual to allow overriding.

public class ParentClass
{
    ...
    public void AddDevice(IDevice device)
    {
        DoSomethingBefore();
        AddDeviceCore(device);
        DoSomethingAfter();
    }

    protected abstract void AddDeviceCore(IDevice device);
    ...

In the subclass now AddDeviceCore needs to be implemented.

public class ParentClass
{
    ...
    protected override void AddDeviceCore(IDevice device)
    {
        DecoderType.Value = device.DecoderName;
        ...
    }
    ...

One other thing is to remark here. In the typical case, that only one step needs to be adjusted, the overridable method is named like the calling method with the suffic Core.
