Developer Tree (aka Solution Tree)

Solution

The Solution consists of these projects:

  • DomainModel
    Contains the (UI-)Domain classes of the Configuration Client

  • ConfigClient.ConsoleApp
    The runner to instantiate the ExcelAdapter.

  • ConfigClient.ExcelAdapter
    The ExcelAdapter is the bridge to configure the Config Client via an Excel sheet.

  • ConfigClient.Sdk
    This project could be treated as an internal SDK for test developers to interact with UI elements (abstracts the UI).

  • ConfigClient.Sdk.Tests
    Unit tests for ConfigClient.Sdk

  • ConfigClient.Api
    API for high level interaction with the Config Client to address use cases.

The disctinction beween the SDK and the API is the level of abstraction. SDK abstracts the UI, API abstracts an use case. To add a device it is one call in the API (Add Device) but it needs different steps of the UI which are triggered via the SDK (Activate Device Pane, Select the device type in tree, setup the device properties).
