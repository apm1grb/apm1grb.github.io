    XML code documentation comments

    Typical phrases, templates and usages

    • reference another type, method, property

      <see cref=""/>

    • summary to describe the purpose of a dialog

      <summary>Dialog to add a new <see cref="..."/>.</summary>

    • summary for a contract

      <summary>Contract to specify the interaction with the maps and logical structure.</summary>

    • summary for a setter

      <summary>Sets the name for the <see cref="UserGroup"/>.</summary>

    • returning itself for fluent interfaces

      <returns>The instance of the <see cref="..."/> itself to foster fluent interface style.</returns>

    • template for boolean return values

      <value><c>true</c> if this instance is login attempt possible; otherwise, <c>false</c>.</value>

    • where to use <seealso cref="..."/> (at the end)

        /// <summary>bla bla bla
  /// <para>foo foo foo.</para>
  /// <seealso cref="ThisClass.ThatMethod"/>
  /// </summary

    • documentation for page objects

        /// <summary>Manages the UI interactions
