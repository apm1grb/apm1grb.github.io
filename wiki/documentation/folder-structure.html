    Folder structure

    The DevTree's doc-folder contains the project`s documentation itself and the configuration files for DocFX to process the content.
    The content and intention in detail:

    • folder doc itself

      • artifacts to configure the DocFX documentation system

        • docfx-json
          the DocFX project file
        • toc.yml
          the table of content file to hook up the wiki-pages
        • index.md
          the landing page for the DocFX generated online documentation

    • sub-folder wiki

      • this is the repository of the Git sub-module from BVMS UI Automation project Wiki

      • it was included via Git´s submodule sub-command

        git submodule add
  git@ssh.dev.azure.com:v3/<organization>/<project>/<repository>.wiki doc/wiki

    • sub-folder sdk

      • contains the metadata for the SDK part of the documentation
      • extracted from projects ConfigClient.DomainModel and ConfigClient.Sdk

    • sub-folder api

      • contains the metadata for the API part of the documentation
      • extracted from projects ConfigClient.Api and ConfigClient.ConsoleApp

    • sub-folder sdk-api

      • artifacts to configure the DocFX documentation system
      • merges content based on SDK and API metadata together

    • sub-folder _site

      • this is the repository of the Git sub-module with the static web-sites as an output of DocFX

      • the repository is connected to the web site hosting platform (e.g. github.io as a remote)

        git submodule add git@<alias>:<username>/<username>.github.io.git doc/_site
