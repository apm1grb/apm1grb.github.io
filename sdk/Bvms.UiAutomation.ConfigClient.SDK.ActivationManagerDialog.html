    Syntax
    public class ActivationManagerDialog : GenericDialog<ActivateExistingConfigurationDialog, ConfigClientPageObject>, IGenericDialog<ActivateExistingConfigurationDialog, ConfigClientPageObject>
    Remarks

    After closing the ActivationManagerDialog a simple "OK/Cancel"-dialog appears. Because that dialog has no deeper meaning we can handle it inside the logic of this class. With a confirmation of this dialog we return the next but one page object and skip the simple "OK/Cancel"-dialog for an outside perspective.

    Examples

    So the initial expected signature

    public class ActivationManagerDialog : GenericDialog<GenericDialog<ActivateExistingConfigurationDialog, ActivationManagerDialog>, ConfigClientPageObject>

    gets simplified to

    public class ActivationManagerDialog : GenericDialog<LoginDialog<ConfigClientPageObject>, ConfigClientPageObject>
